一、研究型：不是写代码，而是省时间

Last30Days-Skill 的登顶故事，可能是 Agent 生态里最反直觉的案例。

它不帮你写一行代码。它的核心功能是——帮你在 Reddit、X、YouTube、Hacker News、Polymarket 五个平台上搜索一个话题，然后 AI 自动合成一份摘要。

就这么一个功能，6月8日 +3,558★，6月10日 +3,177★，6月11日 +2,561★——三天加起来近 10,000 颗星，连续三天霸占 GitHub Trending 第一。

为什么研究型 Agent 是蓝海？

🎯 编码 Agent 已经卷到天花板了——Claude Code、Codex、Cursor、Goose……每个都在比谁写的代码更好。但"帮人研究"这个场景，几乎没有像样的 Agent 产品。

🎯 研究是所有知识工作者的刚需——产品经理要调研竞品，研究生要看论文，创业者要分析市场。但直到今天，大部分人还在手动开五个标签页。

🎯 跨平台搜索有明确的壁垒——每个平台的 API、登录、反爬都不一样。Last30Days-Skill 同时覆盖 Reddit + X + YouTube + HN + Polymarket，这就是护城河。

研究型路线的核心逻辑： Agent 最大的价值不是替代你的创造力，而是替代你的信息获取时间。在信息爆炸的时代，谁帮用户省下最多搜索时间，谁就掌握了最大的市场。

二、方法论型：定义"怎么写好 Agent"

与研究型不同，Superpowers 要解决的是另一个问题：Agent 开发的工程方法论。

写一个能用的 Agent 很简单——写一个"好"的 Agent 很难。Superpowers 不仅是工具，更是一套"怎么写好 Agent"的方法论。它定义了 Agent 技能的标准格式、开发流程、测试方法和部署规范。

它的战绩是惊人的：223,565 颗星，7 次登上 GitHub Trending。这个数据说明它不是流星，而是 Agent 生态里的基础设施。

方法论派的两员大将

今天榜单上还有另一个方法论派的代表：addyosmani/agent-skills（+781★，51.6K★）。这是 Google Chrome 工程总监 Addy Osmani 亲自维护的项目，提供"Production-grade"（生产级）的 Agent 工程技能。

Superpowers 定义"技能应该怎么写"，Agent-Skills 提供"直接能用的生产级技能"——两者路线相似但定位互补。

方法论路线的核心逻辑： Agent 生态的瓶颈不是 Agent 不够多，而是 Agent 质量参差不齐。谁定义了"好 Agent"的标准，谁就掌握了 Agent 生态的规则制定权。

三、品类化型：Agent 的"App Store"时刻

PM-Skills 可能是三条路线里最意味深长的那个。

它不做研究，不搞方法论——它做的是产品经理的 Agent 技能包。100+ 个技能，覆盖竞品分析、用户访谈、需求排期、PRD 撰写、路线图规划……几乎 PM 岗位的所有流程化工作。

连续两天上榜：6月10日 +808★，6月11日 +775★。这不是一个偶然爆发的项目——它在验证一个模式。

品类化的想象力

🔥 「产品经理需要自己的 Agent」——这个需求被验证了。PM-Skills 的火爆说明开发者以外的职业角色同样渴望 Agent 工具。

🔥 那设计师呢？市场呢？HR 呢？财务呢？——PM-Skills 打开了一扇门。每个职业角色都有可能拥有自己的 Agent Skill 包。

🔥 100+ 技能的数量壁垒：不是做几个提示词就完事——100+ 个结构化技能的积累，本身就是难以复制的护城河。

品类化路线的核心逻辑： Agent 技能的未来不是"一个通用 Agent 做所有事"，而是"每个职业都有自己的 Agent 工具包"。就像 App Store 不是只有一款应用——品类化才是真正的规模化的起点。

四、谁会赢？

第一，三条路线不互斥，它们在补全 Agent 生态的不同层

研究型解决"Agent 能做什么"，方法论解决"Agent 怎么做得好"，品类化解决"Agent 给谁用"。这三者更像是一个生态的三个支柱，而不是你死我活的对手。一个成熟的 Agent 生态，需要同时有优秀的应用、可靠的框架和广泛的受众。

第二，品类化的长期想象力最大

研究型和方法论都有天花板：研究工具再强，也只是信息获取这一环节。方法论再完善，服务的依然是开发者群体。但品类化打开了 Agent 从「开发工具」走向「全民工具」的大门。今天 PM-Skills 验证了 PM 群体的需求，明天就可能验证设计师、市场人、教师的 Agent 需求。

第三，方法论可能成为品类的"操作系统"

如果 Superpowers 定义的"技能格式"成为行业标准，那品类化的 Agent 技能都会基于这个标准来构建。届时，Superpowers 会变成 Agent 技能生态的"Android"——不自己做应用，但所有应用都跑在上面。方法论 + 品类化的组合，可能是终局答案。

🐒 猿哥点评

过去半年，所有人都盯着 Agent 写代码的能力——Claude Code 多强、Codex 多快、Cursor 多智能。这当然重要，但它让 Agent 生态变成了一个"窄市场"：只有程序员关心。

这周的榜单第一次让我觉得 Agent 的"iPhone 时刻"可能要来了。不是因为技术突破，而是因为使用者不再只是程序员。

研究型、方法论、品类化——三条路线并行不悖。真正值得关注的是：当 PM 都在用 Agent 技能包的时候，Agent 就已经不再是"开发工具"了。它是"生产力工具"。
