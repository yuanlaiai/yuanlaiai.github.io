## 📋 分镜脚本

---

### Part 1：Hook（0:00-0:45）

**画面：** 快速剪辑——猿哥在终端敲命令 / GitHub Trending 页面刷新 / 网站自动更新 / 公众号草稿箱出现新文章

**旁白（语速快，带情绪）：**
> 三个月前，我做了一个决定：让 AI 替我做我每天最繁琐的工作——刷 GitHub Trending，写分析，更新网站。
> 我花了大概一个周末，搭了一套全自动流水线。
> 然后我连续观察了 30 天。
> 今天想跟你聊聊：AI 到底能不能替代一个内容创作者？
> 答案是……能，也不能。

**画面：** 屏幕黑底白字打出标题

---

### Part 2：为什么要做这个（0:45-2:00）

**画面：** 猿哥出镜 + GitHub Trending 页面

**旁白：**
> 我叫猿来AI，每天做的事情很简单：刷 GitHub Trending，挑 5 个跟 AI 相关的项目，写深度分析，更新到网站。
> 听起来不难是吧？
> 但每天都要做。选题、阅读 README、查资料、写分析、更新数据、发文章。
> 重复劳动占了我 60% 的时间。
> 我就想：这些能不能让 AI 干了？

**画面：** 流程图动画——人工 vs AI 流程对比

**旁白：**
> 于是我开始搭这套流水线。

---

### Part 3：动手搭建（2:00-5:00）

**画面：** 终端 + VS Code 操作实录，配打字动画音效

**旁白：**
> 第一步：爬数据。
> GitHub Trending 没有官方 API，所以用 Headless Browser 定时抓取。

**画面：** 代码展示——browser_navigate → extract articles → parse JSON

```python
# 核心就这几行
browser.navigate("github.com/trending")
projects = extract_all_articles()
```

**旁白：**
> 第二步：让 AI 分析。
> 把抓到的项目标题、描述、Star 数喂给大模型。
> 我用的不是「帮我写分析」这种模糊指令。
> 而是精确告诉它：你的读者是 AI 开发者，关注实用性和趋势判断，用短句，每段不超过 3 行。

**画面：** Prompt 对比——❌ 模糊指令 vs ✅ 精确指令

**旁白：**
> 第三步：自动化发布。
> 生成的 JSON → 自动更新网站 → 发到公众号草稿箱。
> 一条命令搞定。

**画面：** 终端执行 `./deploy.sh` 的实时输出

---

### Part 4：反转来了（5:00-7:30）

**画面：** 节奏放缓，猿哥出镜

**旁白：**
> 头一周，效果惊艳。
> 每天自动出 5 篇分析，数据准确、格式工整、按时发布。
> 我心想：成了。以后躺着就行。

**画面：** 监控面板——数据飙升

**旁白：**
> 但两周后我发现一个问题。
> 同一个项目类型反复出现——Agent 框架、Token 压缩、记忆引擎。
> AI 判断「热门」的标准非常单一：Star 数增长快 = 好项目。
> 但读者真的每天都要看 Agent 框架吗？

**画面：** AI 选出的项目列表（全是 Agent 相关）vs 猿哥手动选出的（混入了一个冷门但有趣的项目）

**旁白：**
> 第三周我做了一个实验：
> 同时让 AI 自动发，和我手动筛选发，对比两周的数据。
> 结果：AI 版的阅读量稳定但平坦。我手动版的偶尔会出现「小爆款」。
> 差别在哪？
> AI 会选「安全」的爆款。我会选「有趣」的项目——哪怕它只有 500 星。

---

### Part 5：技术展示（7:30-9:30）

**画面：** 系统架构图动画

**旁白：**
> 这是我的全套流水线架构。全部开源，代码在 GitHub。

**画面：** 架构图逐步展开

```
GitHub Trending
    ↓ (Headless Browser)
项目数据
    ↓ (LLM 分析)
分析文章 + JSON
    ↓ (自动构建)
GitHub Pages 网站更新
    ↓ (公众号 API)
草稿箱待发布
```

**旁白：**
> 用了哪些技术？

| 环节 | 工具 | 说明 |
|------|------|------|
| 数据爬取 | Browserbase + Headless | 每天自动抓 Trending |
| AI 分析 | DeepSeek / Claude | 写深度分析 |
| 网站托管 | GitHub Pages | 静态站点，零成本 |
| 自动发布 | WeChat API | 公众号草稿箱直达 |

---

### Part 6：猿哥观点 & CTA（9:30-11:00）

**画面：** 猿哥出镜，背景是网站界面

**旁白：**
> 30 天实验结束了。我的结论是：
>
> **AI 能干 80% 的活，但剩下的 20% 是灵魂。**
>
> 它可以帮你收集信息、整理数据、生成初稿。
> 但它不知道什么项目「有意思」。
> 它不知道你的读者今天想看什么。
> 它更不知道什么内容会让你在屏幕前「哇」出来。
>
> 所以我现在的工作流是：
> AI 替我干粗活——刷数据、写初稿、排版发布。
> 我干细活——选题、定调、加观点。
>
> 以前我每天花 2 小时做日报。
> 现在花 20 分钟。
> 省下来的时间，用来想更好的选题。

**画面：** 文字浮现

```
AI 替你干活
你替读者活着
```

**旁白：**
> 这就是我在 Build in Bilibili 分享的「AI 创造」。
> 不是用 AI 替代人，而是用 AI 放大人的价值。
>
> 这套流水线的源码和教程，我放评论区了。
> 我是猿来AI，关注我，每天给你扒一个 GitHub 热门 AI 项目。
> 下期见 🐒
