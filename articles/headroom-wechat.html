一、你的Agent，一半的钱白花了

先想一个问题：你每次让 AI Agent 调一个工具、读一个文件、查一段日志，发生了什么？Agent 把工具输出的全部内容——包括格式、日志、元数据、冗余信息——一股脑塞给 LLM。问题是：LLM 按 Token 计费。

说人话就是：你花 100 块钱的 Token 费，60 到 95 块是白花的。

二、headroom 在做什么

一句话概括：在数据到达 LLM 之前，先压缩。不是简单粗暴地截断或丢弃。headroom 使用专门的压缩算法——保留语义关键信息，去掉冗余噪音——保证压缩后的内容，LLM 仍然能给出和原始内容一致的答案。

官方数据：60-95% 更少 Token，同样答案。

这意味着什么？

✅ 每月花 1 万在 Token 上 → 只需 500 到 4000 块

✅ Agent 响应速度大幅提升，不再受上下文长度拖累

✅ 不再频繁触发上下文窗口限制，规模化不再是问题

三、三种用法，适配所有场景

① 作为 Python 库

pip install headroom
from headroom import compress

compressed = compress(tool_output)
# 丢给 LLM 的是压缩后的内容

一行代码嵌入现有代码，最轻量的使用方式。

② 作为代理服务器

配置一个 Headroom Proxy，所有经过的 LLM 请求自动被压缩拦截。不需要改任何代码。适合已有完整工作流的团队，零侵入。

③ 作为 MCP Server

所有兼容 MCP 的 Agent 可以即插即用。Claude Code、Codex、Cursor……一次配置，所有 Agent 受益。
